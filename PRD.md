# PRD - Generate Image MVP

## 1. 文档信息
- 项目名称：电商宣传图生成 MVP
- 版本：v1.0 (冻结版)
- 冻结日期：2026-03-09
- 目标验收日期：2026-03-12
- 开发路径：`/Users/fishyuu/idea_project/mul_p2p/generate_image`

## 2. 产品目标
- 一句话目标：基于新品图自动检索爆款参考并生成宣传图，先跑通可用 MVP。
- 成功标准：端到端流程可稳定执行，成功率 `>= 70%`。
- 成功率公式：`点赞数 / (点赞 + 点踩 + 未反馈)`。

## 3. 用户与使用场景
- 目标用户：中国跨境电商相关用户（当前仅本地开发验证）。
- 主要场景：
1. 从新品库选图或上传新品图。
2. 系统检索爆款图（Top3）。
3. 系统分析风格并选 1 张参考图。
4. 生成 1 张宣传图并展示。
5. 用户点赞/点踩并填写优化意见（点踩必填 >= 10 字）。

## 4. 数据范围与来源
- 爆款库路径：`/Users/fishyuu/idea_project/mul_p2p/fashion_img`
- 新品库路径：`/Users/fishyuu/idea_project/mul_p2p/product`
- 自动生成数据文件：
1. `products.csv`：由爆款库自动生成。
2. `new_products.csv`：由新品库自动生成。
- ID 规则：
1. `product_id` = 图片文件名去后缀。
2. `new_id` = 图片文件名去后缀。
3. 同名冲突时自动加目录前缀（例如 `dirA_xxx`）。

## 5. 功能需求（MVP）

### FR-001 数据导入与清洗
1. 扫描爆款库生成 `products.csv`。
2. 扫描新品库生成 `new_products.csv`。
3. 允许缺失字段补齐：
- `sales_count=0`
- `price=0`
- 缺失文本字段可空或 `unknown`（以实现为准，但必须可追踪）。
4. 元数据补齐策略：优先用 Qwen 自动补齐；失败时按默认值继续导入。

### FR-002 去重
1. 去重范围：仅爆款库。
2. 去重方式：按图片 `sha256` 完全重复。
3. 保留规则：保留第一条记录。
4. 去重时机：写入 Milvus 前执行。
5. 日志文件：`/Users/fishyuu/idea_project/mul_p2p/generate_image/logs/dedup.log`

### FR-003 检索与参考图选择
1. 检索策略：Dense + BM25 混合检索。
2. 权重固定：Dense:Sparse = `7:3`。
3. 不做类目约束。
4. 检索返回：Top3。
5. Qwen 对 Top3 完成两件事：
- 生成 `style_prompt`
- 选出最终 1 张参考图
6. Qwen 输出必须为固定 JSON；格式异常则回退为检索最高分参考图。

### FR-004 相似度合规告警
1. 计算对象：新品图 vs 检索 Top3。
2. 指标：embedding cosine score。
3. 阈值：`>= 0.7` 触发告警。
4. 触发后动作：直接终止任务并报错，不允许强制继续。
5. 告警事件必须写审计记录。

### FR-005 图像生成
1. 每次任务默认生成 1 张图。
2. 目标比例支持：`1:1`、`3:4`、`4:1`、`9:16`。
3. 最大时延：单任务 `<= 120s`。
4. 失败定义：接口报错。
5. 重试策略：最多 1 次重试（总尝试 2 次）。
6. 重试方式：Qwen 自动微调 prompt（构图/光线/贴合度/背景）。
7. 二次失败：任务状态 `failed`，返回已有结果（若存在）。

### FR-006 存储与历史
1. MinIO 本阶段必须接入。
2. 上传对象：原图、新品图、参考图、生成图。
3. 预签名 URL 过期时间：`60s`。
4. 历史聊天记录必须保存。
5. 审计字段必须保存（见 `BACKEND_STRUCTURE.md`）。

### FR-007 前端页面
1. `/studio`：生成工作室（核心）。
2. `/gallery`：历史画廊与下载。
3. `/metrics`：指标页（总任务数、成功率、点踩率）。
4. 无登录系统，使用浏览器 `session_id` 做会话隔离。
5. 实时进度采用 SSE。

## 6. 非功能要求
1. 配置必须独立文件化，支持本地/生产切换。
2. Key 未就绪时允许 mock 模式；填入 Key 后切换真实调用。
3. 所有关键流程必须有日志与审计可追溯。

## 7. 不在本期范围
1. 账号登录/JWT/RBAC。
2. 多模型切换 UI。
3. Redis/Celery 异步队列。
4. 计费与充值。
5. 商业法务完整合规体系（仅保留告警与审计能力）。
