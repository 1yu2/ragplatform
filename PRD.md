# RAG 管理系统产品需求文档 (PRD)

## 1. 文档信息
- 项目名称: `rag_sth`
- 文档版本: `v1.0`
- 文档日期: `2026-03-04`
- 目标验收日期: `2026-03-10`
- 目标用户: 统计专业人员

## 2. 背景与目标
### 2.1 背景痛点
- 当前需要处理 PDF 文档，无法稳定完成版面级解析（文字、表格、图片等）。
- 问答结果与原文相关性不稳定，缺少可量化评估闭环。

### 2.2 核心目标
- 构建一个可运行的中文 RAG 管理系统，覆盖:
1. PDF 上传与解析入库。
2. 混合检索问答（向量 + BM25）。
3. 可追溯引用展示（页码与段落定位）。
4. 基于 RAGAS 的评估与趋势展示。

## 3. 范围定义
### 3.1 本期范围 (MVP)
1. 仅支持 PDF 文件上传与处理。
2. 版面解析支持: 标题、段落、表格、图片、页眉页脚、脚注、目录。
3. 页眉页脚默认过滤，不参与检索。
4. 中文问答。
5. 问答支持流式输出，首字响应时间 <= 10 秒。
6. 评估集自动构建 50 条，人工抽检 10%。

### 3.2 明确不做 (本期排除)
1. 权限系统与登录。
2. 多租户隔离。
3. 文档版本管理与回滚。
4. 图片深度视觉理解（本期仅 OCR 文本 + 元数据入库）。
5. 专门公式解析（本期明确不支持）。
6. 重排序模型（reranker）本期不接入。

## 4. 用户与关键路径
### 4.1 用户角色
- 主要用户: 统计专业人员。
- 次要用户: 无。

### 4.2 核心路径
1. 用户上传 PDF 到知识库。
2. 系统调用版面解析接口提取结构化内容。
3. 系统做语义切分，生成 chunk，并向量化写入 Milvus。
4. 用户发起提问。
5. 系统按规则决定是否进行问题改写。
6. 系统执行混合检索并融合打分。
7. 系统根据阈值判断是否拒答。
8. 系统调用 LLM 流式生成答案，展示引用与页码。
9. 用户在评估页面查看 RAGAS 指标与趋势。

## 5. 功能需求
## 5.1 文档管理
1. 支持上传 PDF，单文件上限 100MB。
2. 上传后按 `SHA256` 去重。
3. 文件物理存储在 MinIO:
- endpoint: `192.168.10.60:9000`
- bucket: `rag_sth`
- 账号配置走环境变量，不写死明文。
4. 文件列表需展示:
- 文件名
- 文件大小
- 上传时间
- 处理状态（排队/处理中/成功/失败）
- 失败标红

## 5.2 版面解析
1. 解析服务:
- `POST http://192.168.10.60:8888/layout-parsing`
- 鉴权: `Authorization: Bearer <TOKEN_FROM_CONFIG>`
2. 入参:
- `file`: 可访问 URL（MinIO URL）或直传文件（接口支持两者）
- `fileType`: `0` (PDF)
3. 输出标准化目标（内部统一 schema）:
- text_blocks
- table_blocks
- image_blocks
- header/footer blocks（用于过滤）
- 每个 block 需携带 `page,x1,y1,x2,y2,text,type`
4. 表格需双轨存储:
- 结构化 JSON（用于检索）
- Markdown（用于页面展示）
5. 图片需入库:
- OCR 文本
- 元数据（页码、坐标、尺寸等）

## 5.3 切分与索引
1. 切分方式: 语义切分（LangChain semantic splitter）。
2. 约束:
- `chunk_size=2000`（中文字符）
- `chunk_overlap=50`（中文字符）
- 不允许句子被拆断。
- 语义相关内容尽量在同一个 chunk。
3. 每个 chunk 必须保留元数据:
- file_id
- file_name
- page
- paragraph_id
- block_type
- chunk_text
- chunk_index
- source_offset

## 5.4 向量化与检索
1. Embedding 服务:
- URL: `http://192.168.10.60:5001/bge_embed`
- model: `bge-m3`
- 维度: `1024`
- 不需要鉴权 header。
2. Embedding 请求格式:
```json
{"model":"bge-m3","input":["hello"]}
```
3. Embedding 响应格式（兼容字段）:
```json
{"data":[{"embedding":[...], "spare_embedding":{"21":"0.22222"}}]}
```
4. 向量库: Milvus。
5. 稀疏检索: 使用 Milvus BM25 能力。
6. 混合融合:
- 分数融合算法: 加权融合
- 权重: 向量:BM25 = `0.7:0.3`
- 召回参数: `topK=5`
7. 本期不使用 reranker，相关接口预留。

## 5.5 问答生成
1. LLM 接口:
- URL: `https://api.deepseek.com/v1/chat/completions`
- model: `deepseek-chat`
- 强制 `stream=true`
- `max_tokens=131072`
2. 问题改写触发:
- 问题较长或语义模糊时触发（规则可配置）。
3. 改写失败策略:
- 回退到原问题继续检索。
4. 拒答规则:
- 若 `top1 向量相似度 < 0.1`，返回固定文案: `无法回答`。
- 同时展示最多 5 条“最接近片段”供用户参考。
5. 引用展示:
- 格式: `[文件名-页码-段落ID]`
- 必须展示引用内容与页码定位。

## 5.6 评估系统
1. 集成 RAGAS 指标:
- faithfulness
- answer_relevancy
- context_precision
- context_recall
2. MVP 阈值:
- faithfulness: `0.7~0.9`
- answer_relevancy: `0.7~0.8`
- context_precision: `0.6~0.8`
- context_recall: `0.7~0.8`
3. 评估数据:
- 自动生成 50 条测试样本。
- 人工抽检 10% 样本并修正。
4. 页面能力:
- 显示单次评估结果。
- 显示历史趋势图。
- 输出“答案整体质量评分” (`0~100`)。

## 6. 非功能需求
1. 性能:
- 问答首字响应 <= 10 秒（流式）。
- 并发问答目标: 20。
2. 可用性:
- 入库任务失败自动重试 3 次。
- 超过 3 次后状态置为失败并前端标红，等待人工处理。
3. 安全:
- 默认内网使用。
- 配置 CORS 白名单。
- 所有密钥从环境变量读取，不入库、不写死。
4. 可运维性:
- 所有运行参数集中在 config（区分 local/prod）。
- 提供开发向操作文档。

## 7. 验收标准
1. 支持上传 PDF 并完成解析、切分、向量化、入库全链路。
2. 支持中文问题流式问答，首字响应 <= 10 秒。
3. 支持引用定位展示（文件名、页码、段落）。
4. 支持“无法回答”拒答机制和最近片段展示。
5. 支持 RAGAS 评估与趋势页面，含四类指标。
6. 交付件完整:
- 可运行系统
- 操作文档
- 评估报告

## 8. 二期规划（冻结，不进入本期）
1. 登录与权限体系。
2. 多知识库与多租户隔离。
3. 文档版本管理。
4. 图片视觉理解（VLM）。
5. reranker 接入与 A/B 对比。
6. Docker 化与云上部署（阿里云）。
