# 实施计划 (IMPLEMENTATION_PLAN)

1. 可运行的 MVP 系统（上传/入库/问答/评估全链路）。
2. 开发操作文档。
3. 一份评估报告。

## 2. 里程碑倒排（2026-03-04 ~ 2026-03-10）
## Day 1 (2026-03-04): 项目骨架与配置打通
交付:
1. 前后端初始化（React + FastAPI）。
2. 配置体系落地（local/prod + env）。
3. SQLite 与 Milvus 基础连接。
4. MinIO 上传下载连通测试。

验收标准:
1. 本地可启动前后端。
2. 能上传 PDF 到 MinIO 并在数据库落记录。

## Day 2 (2026-03-05): 入库流水线 MVP
交付:
1. SHA256 去重。
2. layout-parsing 接口接入。
3. 版面块标准化与页眉页脚过滤。
4. 语义切分（chunk_size=2000, overlap=50）。

验收标准:
1. 上传后可看到任务状态流转。
2. 解析与切分结果可预览。

## Day 3 (2026-03-06): 检索链路打通
交付:
1. embedding 接口接入（bge-m3）。
2. Milvus 稠密+稀疏写入。
3. 混合检索与加权融合（0.7:0.3）。
4. 拒答阈值逻辑（top1 < 0.1）。

验收标准:
1. 已入库文档可检索返回 topK=5。
2. 低相关问题触发 `无法回答`。

## Day 4 (2026-03-07): 问答与前端聊天
交付:
1. 问题改写策略（长问题/模糊问题触发）。
2. DeepSeek 流式问答接入。
3. 聊天页展示答案 + 引用 `[文件名-页码-段落ID]`。
4. 无法回答场景展示最近 5 条片段。

验收标准:
1. 聊天页端到端可用。
2. 首字响应时间 <= 10 秒（典型查询）。

## Day 5 (2026-03-08): 评估框架
交付:
1. 自动生成 50 条评估样本。
2. 人工抽检入口（10%）。
3. RAGAS 四指标计算。
4. 评估页指标卡与趋势图。
5. 答案整体质量分（0~100）。

验收标准:
1. 可触发评估任务并查看结果。
2. 评估历史可查询并可视化。

## Day 6 (2026-03-09): 稳定性与文档
交付:
1. 失败重试（3 次）与失败标红。
2. 关键日志与错误处理完善。
3. 开发操作文档（安装、配置、运行、评估、排障）。
4. 性能与回归测试。

验收标准:
1. 上传、问答、评估主流程稳定无阻断。
2. 文档可指导新开发者完成部署运行。

## Day 7 (2026-03-10): 验收与交付
交付:
1. 最终演示脚本。
2. 评估报告（指标结果 + 问题清单 + 下一步优化建议）。

验收标准:
1. 满足 PRD 的 MVP 验收项。

## 3. 工作分解 (WBS)
1. 基础设施:
- 项目初始化
- 配置分层
- DB/向量库初始化
2. 入库链路:
- 上传与去重
- 解析接入
- 切分与索引
3. 检索问答:
- 混合检索
- 拒答逻辑
- 流式回答与引用
4. 评估体系:
- 测试集构造
- RAGAS 运行
- 趋势展示
5. 质量保障:
- 单测
- 集成测试
- 性能验证
6. 文档交付:
- 运行文档
- 配置说明
- 评估报告

## 4. 关键验收指标
1. 首字响应时间 <= 10 秒（流式）。
2. faithfulness: 0.7~0.9。
3. answer_relevancy: 0.7~0.8。
4. context_precision: 0.6~0.8。
5. context_recall: 0.7~0.8。

## 5. 风险与应对
1. 风险: 外部服务不稳定（解析/LLM/embedding）。
- 应对: 重试 + 超时 + 降级提示。
2. 风险: 指标不达标。
- 应对: 优先调整切分策略与召回参数。
3. 风险: 工期紧（6 天）。
- 应对: 严格冻结二期需求，优先端到端可跑通。

## 6. 每日检查清单
1. 是否阻塞主链路（上传->问答->评估）？
2. 是否出现新增范围蔓延？
3. 是否有关键参数漂移（topK/阈值/chunk）？
4. 是否有明文密钥进入代码或日志？
5. 是否保持文档与实现同步？

1. 后端源码（FastAPI）。
