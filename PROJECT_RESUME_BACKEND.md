# 项目经历（后端）- RAG 管理系统

## 项目概述
- 项目名称：RAG 管理系统（`rag_sth`）
- 项目周期：2026.03 - 2026.03（MVP）
- 项目角色：后端开发（主导核心链路实现）
- 项目目标：面向中文 PDF 场景，完成“文档入库 -> 混合检索 -> 流式问答 -> 质量评估”闭环，支持引用可追溯与任务可观测。

## 技术栈（后端）
- Python 3.11、FastAPI、HTTPX
- Milvus（稠密向量 + sparse 字段）、SQLite、MinIO
- pypdf（PDF 页级切分）
- 外部模型服务：Paddle 版面解析、bge-m3 Embedding、DeepSeek Chat（SSE 流式）

## 核心工作
1. 负责文档入库全链路设计与实现：PDF 上传、SHA256 去重、MinIO 持久化、异步任务编排、解析/切分/向量化/入库。
2. 实现复杂文档解析容错机制：大文档分批解析（默认 512 页）、失败自动二分降级、单页失败可跳过、页覆盖率校验，显著提升长文档可处理性。
3. 设计并落地中文分块策略：基于语义切分 + 小块合并；补充无标点超长文本硬切分兜底，避免超长 chunk 影响索引稳定性。
4. 打通向量检索链路：接入 bge-m3（1024 维）并写入 Milvus，支持 dense/sparse 字段兼容。
5. 实现混合召回与融合排序：向量召回 + 词法补分融合，支持 `topK=5`、`0.7/0.3` 权重配置化。
6. 实现问答策略层：长问题/模糊问题自动改写、低相关拒答（`top1<0.1`）、引用回溯（文件名-页码-段落ID）。
7. 实现 SSE 流式问答接口：支持 token 级增量返回，异常场景自动回退并落库聊天日志。
8. 建立配置与可运维能力：多环境配置（local/prod）+ 环境变量映射，关键阈值（检索、切分、重试、覆盖率）可热调整。
9. 完成稳定性优化：embedding 批量重试退避、任务状态机（QUEUED/RUNNING/SUCCESS/FAILED）与失败重试（最多 3 次）。
10. 修复 Milvus 文本长度边界问题：将 `chunk_text` 截断由“按字符”改为“按 UTF-8 字节”，并增加截断告警，解决中文多字节导致的 `varchar max_length` 入库失败。

## 项目亮点
- 端到端打通 RAG MVP 后端能力，覆盖上传、入库、检索、生成、日志闭环。
- 面向大 PDF 场景实现分片 + 自适应降级策略，降低外部解析服务波动对主链路影响。
- 关键策略参数全部配置化（chunk、权重、阈值、解析批次、覆盖率），便于线上调优与快速回归。
- 引用与日志可追踪，支持定位到具体文档页码与段落，提升结果可解释性。

## 产出与结果
- 交付可运行后端服务与开发运行手册，支持新环境快速部署联调。
- 主链路具备失败重试与降级能力，能够在外部服务异常时保持可恢复。
- 评估模块已完成任务链路与结果落库框架，当前为 MVP 占位实现（后续可替换为真实 RAGAS 计算）。
