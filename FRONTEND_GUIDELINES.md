# 前端实现规范 (FRONTEND_GUIDELINES)

## 1. 目标
前端负责两类能力:
1. 管理能力: 文件上传、解析状态、切分预览、评估管理与趋势展示。
2. 问答能力: 流式聊天、引用展示、无法回答兜底。

## 2. 技术约束
1. 技术栈: React + Vite + TypeScript + Ant Design。
2. 图表: ECharts。
3. 状态管理: Zustand（轻量）或 Redux Toolkit（二选一，默认 Zustand）。
4. 网络请求: axios + SSE/Fetch stream 处理。
5. 文案语言: 中文。

## 3. 页面清单（MVP 必须）
1. `文件上传页`
2. `文档列表页`
3. `解析结果预览页`
4. `聊天问答页`
5. `评估页`（含历史趋势图）
6. `系统配置页`（只读脱敏展示）

## 4. 路由建议
1. `/kb/upload`
2. `/kb/files`
3. `/kb/files/:fileId/preview`
4. `/chat`
5. `/evaluation`
6. `/settings`

## 5. 核心交互规范
## 5.1 上传流程
1. 限制扩展名: `.pdf`。
2. 限制大小: <= 100MB。
3. 上传后立即显示任务状态。
4. 若命中去重，提示“文件已存在”并跳转到已有记录。

## 5.2 文档列表
字段至少包含:
1. 文件名
2. 大小
3. SHA256（可折叠显示）
4. 上传时间
5. 状态（QUEUED/RUNNING/SUCCESS/FAILED）
6. 操作（预览、重跑）

状态颜色:
1. `SUCCESS` -> 绿色
2. `RUNNING` -> 蓝色
3. `QUEUED` -> 灰色
4. `FAILED` -> 红色（必须高亮）

## 5.3 解析结果预览
1. 按页展示版面块。
2. 表格展示 markdown 版本。
3. 每个块显示:
- block_type
- page
- bbox
- 原始文本
4. 支持查看切分结果（chunk 列表）。
5. 必须同时展示“原始 Markdown”和“渲染结果”双栏，便于快速定位渲染问题。
6. 必须提供 `layout-debug` 调试入口（按文件触发），用于核对页数、结构、块数。

## 5.4 聊天问答
1. 必须流式显示答案。
2. 首字响应时间需在 UI 上可观测（用于调试）。
3. 每条回答必须显示引用信息:
- 格式 `[文件名-页码-段落ID]`
4. 当触发拒答时:
- 固定显示 `无法回答`
- 同时显示最多 5 条最近片段
5. 必须支持聊天历史列表，可点击回看历史问答详情。
6. 必须支持删除单条历史与清空历史（需二次确认）。

## 5.5 评估页
1. 支持启动评估任务。
2. 展示本次评估四类指标:
- faithfulness
- answer_relevancy
- context_precision
- context_recall
3. 展示答案整体质量分（0~100）。
4. 展示历史趋势图（按时间序列）。
5. 标识是否通过阈值区间。

## 6. API 对接约定
统一返回格式:
```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

错误返回:
```json
{
  "code": 1001,
  "message": "error detail",
  "data": null
}
```

## 7. 组件设计建议
1. `FileUploadPanel`
2. `TaskStatusTag`
3. `ChunkPreviewTable`
4. `CitationList`
5. `ChatStreamPanel`
6. `MetricsCardGroup`
7. `TrendChart`

## 8. 性能与体验
1. 聊天流式渲染应支持 token 增量追加。
2. 长列表使用虚拟滚动（文档/chunk 列表）。
3. 失败场景必须可重试，不允许静默失败。
4. 所有异步操作必须有 loading 状态。
5. 大文档预览默认只渲染选中页，避免一次渲染全部页导致卡顿。

## 9. 代码规范
1. TypeScript 严格模式开启。
2. API schema 使用 zod 校验（建议）。
3. 页面状态与业务状态分离。
4. 时间、分数、状态格式化函数集中维护。

## 10. 安全规范
1. 前端不得存储明文 API key。
2. 配置页只展示脱敏值。
3. 默认允许内网来源访问，跨域由后端白名单控制。

## 11. MVP 验收点
1. 从上传到入库状态更新可全链路可视化。
2. 聊天页面流式输出稳定，首字 <= 10 秒。
3. 每次回答都能查看引用片段与定位信息。
4. 评估页可看到指标与趋势图。

## 12. Markdown 渲染规范（新增）
1. 预览页与聊天页必须复用同一套 Markdown 渲染工具，避免展示不一致。
2. 表格需兼容 `|` 与 `｜` 两种分隔符。
3. 表格列数不一致时前端要容错（自动补齐/截断），不能出现整表错位。
4. 单元格内至少支持 `**粗体**`、`*斜体*`、链接与行内代码。
5. 引文角标 `[1]` 需渲染为上标，并支持点击展开对应引用原文。
